## Q1. 모바일 레이아웃 — 탭바 구성

현재: 상단 TopBar (로고 + 설정 기어 아이콘)
스펙: 모바일은 하단 탭바

- **A안**: 탭바 없이 TopBar 유지 + 모바일 크기만 최적화

- **C안**: 지도 / 업로드 / 설정 3탭

추천: A안 — 현재 기능이 단일 화면(지도) 중심이라 탭으로 나눌 콘텐츠가 없음. 설정은 기어 아이콘 메뉴로 충분. 탭바를 넣으면 모바일에서 지도 세로 공간만 줄어듦. Phase 2에서 사이드 패널/통계 탭이 생기면 그때 도입.

## Q2. UploadOverlay 모바일 처리

현재 `w-[460px]` 고정 너비 → 모바일에서 잘림.
모바일에서 드래그앤드롭 불가 → 버튼 중심으로 전환 필요.

- **A안**: 반응형 처리 (모바일: 전체 너비, 드래그 문구 숨김 / PC: 현재 유지)
- **B안**: 모바일 전용 별도 컴포넌트
- **C안**: 모바일·PC 동일하게 전체 리디자인

## Q4. 삼성 인터넷 / 미지원 브라우저 대응

삼성 인터넷은 `showDirectoryPicker` 미지원. 안드로이드 점유율 높음.

- **A안**: UA 감지 → 안내 모달 ("Chrome 브라우저를 사용해 주세요")
- **B안**: `showDirectoryPicker` 지원 여부로 감지 (UA보다 정확)
- **C안**: Sprint 5 범위 제외, fallback 없이 에러 처리만

추천: B안 — UA 문자열은 위양성 가능. `typeof window.showDirectoryPicker === 'function'` 체크가 정확하고 미래 브라우저에도 대응. 미지원 시 "이 브라우저는 폴더 선택을 지원하지 않습니다. Chrome을 사용해 주세요." 모달 표시.
