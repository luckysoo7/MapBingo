### Q1. 폰에서 어떤 브라우저로 접속하나?

추천: Chrome — 삼성 인터넷도 showDirectoryPicker 지원하지만 동작이 다를 수 있음.
확인 방법: 브라우저 주소창에 `chrome://version` 입력 → 버전 번호 알려줘.

> 답변:146.0.768.0164

### Q2. 카메라 폴더에 파일이 대략 몇 개?

사진뿐 아니라 mp4, gif, 스크린샷 등 전체 합산.
showDirectoryPicker는 확장자 구분 없이 폴더의 **모든 파일**을 열거하므로,
mp4 200개 + jpg 3000개면 3200개를 전부 읽는다. 이것이 freeze의 주요 원인.

> 답변: 200GB, 33156Files.

### Q3. USB 디버깅 가능한가?

노트북 Chrome에서 `chrome://inspect`로 폰의 탭을 원격 디버깅할 수 있음.
가능하면 정확한 에러 메시지와 성능 프로파일을 볼 수 있어서 진단이 훨씬 정확해짐.

준비 방법:

1. 폰: 설정 > 개발자 옵션 > USB 디버깅 ON
2. USB로 노트북에 연결
3. 노트북 Chrome: `chrome://inspect` 접속
4. 폰의 mapbingo.vercel.app 탭이 보이면 "inspect" 클릭

추천: 가능하면 해보자 — 이거 한 번이면 정확한 원인 특정 가능.

> 답변: 며칠 뒤에는 가능한데, 지금 당장은 안돼. 원격데스크톱이라 usb 연결 불가.

### Q4. 아래 즉시 수정안 먼저 테스트할까?

질문 답변 기다리지 않고 바로 적용할 수정:

- **"사진 선택하기" 버튼을 항상 1순위로 표시** (input type file = 네이티브 갤러리)
- "폴더 선택하기"는 2순위 작은 링크로 변경 (원하는 사람만)
- 이렇게 하면 showDirectoryPicker를 안 거치므로 freeze 자체가 발생하지 않음

추천: Y — 이 수정만으로 모바일 문제 80% 해결 가능. 나머지 20%는 Q3 디버깅으로 정밀 진단.

> 답변: 그래 위 답변 고려하고, 일단 너 제안대로 해야지 .
